# 4학년 교과 어휘 학습 앱

초등학교 4학년 학생이 **국어 · 사회 · 과학** 교과서의 핵심 어휘를
플래시카드 → 퀴즈 → 오답 복습 흐름으로 익히는 학습 웹앱입니다.

React + Vite + TypeScript로 만들었고, **Supabase**(PostgreSQL)로 학생별 로그인과
학습 기록(오답·정답률)을 저장합니다. 음성은 표준 Web Speech API를 사용합니다.

- **학생 로그인**: 아이디 + 비밀번호로 가입/로그인 (이메일 불필요)
- **학생별 기록**: 오답·정답률이 계정에 저장되어 어떤 기기에서 로그인해도 유지
- **관리자(선생님)**: PIN 번호로 학생 목록 확인 및 계정 삭제

---

## 1. 실행 방법

필요 도구: [Node.js](https://nodejs.org) 18 이상

```bash
npm install      # 의존성 설치 (처음 한 번)
npm run dev      # 개발 서버 실행 → 안내된 주소 접속
npm run build    # 배포용 빌드 (dist/ 폴더 생성)
npm run preview  # 빌드 결과 미리보기
```

> ⚠️ 로그인·저장 기능을 쓰려면 먼저 **2. Supabase 설정**을 끝내고
> `.env` 파일에 접속 키를 넣어야 합니다. 키가 없으면 로그인 화면에
> "서버 접속 정보가 설정되지 않았습니다" 안내가 표시됩니다.

---

## 2. Supabase 설정 (최초 1회)

1. [supabase.com](https://supabase.com)에서 무료 프로젝트를 만듭니다.
2. 좌측 메뉴 **SQL Editor → New query**에 `supabase/schema.sql` 내용을 모두
   붙여넣고 **Run**을 누릅니다. (테이블 3개 + 함수 + 보안 정책 생성)
3. **Project Settings → API**에서 다음 두 값을 복사합니다.
   - `Project URL`
   - `anon public` 키
4. 프로젝트 루트에 `.env` 파일을 만들고(`​.env.example` 복사) 값을 채웁니다.

   ```bash
   VITE_SUPABASE_URL=https://xxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGciOi...
   ```

5. `npm run dev`로 다시 실행하면 로그인/회원가입이 동작합니다.

> 🔐 비밀번호는 평문이 아니라 bcrypt 해시로 저장되며, 검증은 서버(Supabase
> 함수) 안에서만 이루어집니다. `students` 테이블은 RLS로 직접 조회가 차단됩니다.

## 3. 주요 기능

| 모드 | 설명 |
|---|---|
| 🔐 로그인/회원가입 | 아이디+비밀번호로 가입·로그인. 로그인 상태는 새로고침 후에도 유지(localStorage 세션) |
| 📇 어휘 카드 | 앞면(어휘·교과·단원·이모지)을 보고 누르면 뒷면(쉬운 뜻·예문)으로 뒤집힘. 이전/다음 이동, 소리로 듣기. 상단에 **오늘의 어휘** 표시 |
| 📝 퀴즈 풀기 | 한 번에 최대 5문제. **뜻 맞히기**와 **문장 빈칸 채우기** 두 유형이 무작위로 출제. 보기 4개(정답1+오답3), 즉시 정답/오답 피드백, 마지막에 결과·정답률 화면 |
| 🔁 오답 복습 | 퀴즈에서 틀린 어휘만 모아서 복습. 소리로 듣기 / 복습 완료 버튼 |
| 🧑‍🏫 관리자 | 로그인 화면 하단 **관리자(선생님용)** → PIN `20260606` 입력 → 학생 목록 확인·삭제 |

- **교과 필터**: 전체 / 국어 / 사회 / 과학 선택 가능 (카드·퀴즈·상태표시에 반영)
- **상태 표시줄**: 선택 교과, 학습 어휘 수, 저장된 오답 수, 누적 정답률
- **TTS(소리로 듣기)**: `[어휘]. [쉬운 뜻]. 예문. [예문]` 순서로 한국어 음성 재생.
  음성 미지원 브라우저에서는 안내 문구 표시
- **빈칸 채우기**: 보기를 클릭하거나 빈칸 위로 드래그 앤 드롭해서 답할 수 있음
- **반응형**: 데스크톱·태블릿·스마트폰 모두 지원 (모바일에서 2열 → 1열)

## 4. 교사가 어휘 데이터를 수정하는 방법

모든 어휘는 한 파일에 모여 있습니다.

> 📄 `src/data/vocab.ts` 의 **`VOCAB_DB`** 배열

어휘 하나를 아래 형태로 추가/수정/삭제하면 됩니다.

```ts
{
  id: "so-10",                                   // 겹치지 않는 고유 값
  word: "축제",                                  // 어휘
  subject: "사회",                               // "국어" | "사회" | "과학"
  unit: "우리 지역의 문화",                       // 단원 이름
  easy_meaning: "여러 사람이 모여 즐기는 행사",   // 쉬운 뜻
  example_sentence: "우리 지역에서는 해마다 봄에 축제가 열려요.", // 예문
  image_url: "🎉",                               // 이모지 1개
}
```

⚠️ **중요**: `example_sentence`(예문) 안에는 `word`(어휘)가 글자 그대로 반드시
들어가야 합니다. 빈칸 채우기 퀴즈가 예문에서 어휘를 찾아 빈칸으로 바꾸기 때문입니다.

파일을 저장하면 개발 서버가 자동으로 화면을 새로 고칩니다. 별도 빌드 없이도 반영됩니다.

## 5. 학습 기록(오답·통계) 저장 방식

학생별 학습 기록은 **Supabase(PostgreSQL)**에 저장되어, 어떤 기기에서 로그인해도
본인의 기록이 그대로 유지됩니다.

| 테이블 | 내용 |
|---|---|
| `students` | 학생 계정 (아이디 · bcrypt 비밀번호 해시 · 가입일) |
| `student_wrong_words` | 학생별 틀린 어휘의 `id` 목록 |
| `student_stats` | 학생별 누적 시도 수 / 정답 수 (정답률 계산용) |

저장 규칙:

- 퀴즈에서 **틀리면** 해당 어휘를 오답 목록에 추가 (같은 어휘는 중복 저장 안 함)
- 퀴즈에서 **다시 맞히면** 오답 목록에서 제거
- 오답 복습 화면에서 **복습 완료**를 누르면 제거
- 학생 계정을 삭제하면 오답·통계도 함께 삭제됩니다 (`on delete cascade`)

> 로그인 세션 자체(누가 로그인했는지)는 브라우저 `grade4_student_v1` 키에
> 보관되어 새로고침해도 로그인이 풀리지 않습니다.

## 6. 관리자(선생님) 기능

1. 로그인 화면 하단 **관리자(선생님용)** 링크를 누릅니다.
2. PIN 번호 **`20260606`** 을 입력합니다. (검증은 Supabase 함수에서 처리)
3. 등록된 학생 목록이 나오며, 각 학생의 **삭제** 버튼으로 계정을 지울 수 있습니다.
   (확인 창 후 삭제되며, 해당 학생의 오답·통계도 함께 삭제됩니다.)

> PIN 번호를 바꾸려면 `supabase/schema.sql`의 `admin_list_students`,
> `admin_delete_student` 함수 안 `'20260606'` 값을 수정한 뒤 SQL Editor에서
> 다시 실행하세요.

## 7. 배포 방법 (Vercel)

저장소에 포함된 `vercel.json`이 새로고침 시에도 정상 동작하도록 설정합니다.

### GitHub 연동 (자동 배포)

1. 이 폴더를 GitHub 저장소에 올립니다.
2. [vercel.com](https://vercel.com)에서 **Add New → Project → Import**로 저장소 선택.
3. 프레임워크가 **Vite**로 자동 감지됩니다.
4. ⚠️ **Environment Variables**에 두 값을 반드시 추가합니다.
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
5. **Deploy**. 이후 GitHub에 푸시할 때마다 자동 재배포됩니다.

> 환경변수를 나중에 추가/변경하면 **Redeploy**를 한 번 해야 반영됩니다.
